# V3 试点页选择标准

V3.0 先做最小闭环样例页，目标是验证 manifest 与 prompt 协议是否够用。

## 优先级

优先选择：
- CRUD 列表页
- Detail 详情页

## 选择标准

- PRD、Figma、API、Model 四项输入齐全
- 页面边界清晰，业务规则不过多
- 至少覆盖一个主接口和一个主模型
- 至少包含一个核心动作，如查询、详情、创建、更新
- 不依赖复杂跨页流程

## 建议场景

- 用户列表页
- 订单列表页
- 商品详情页

## 暂不优先

- 强工作流页面
- 大量联动表单页面
- 高度自定义可视化页面
- 依赖多个后端域的大聚合页面
